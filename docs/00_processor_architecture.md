# 处理器总体架构规格

> 文档状态：v0.1 草案  
> 当前阶段：项目目标与首版边界  
> 原则：先冻结整体架构，再进入模块接口和 RTL 实现。

## 1. 项目定位

本项目设计一颗可综合、可验证、可完成 ASIC 前端与综合后时序分析的 32 位 RISC-V 处理器核。处理器采用单发射、顺序执行、五级流水微架构，当前主线是掌握同步处理器的数据通路、流水控制、验证方法以及综合和 STA 闭环。

项目首先是一项学习型工程，其次才是功能实现。每项架构能力都应具备以下证据：

- 可以解释其架构语义和逐周期行为；
- 具有定向测试、关键波形和必要的断言；
- RTL 可综合，不依赖仅适用于仿真的写法；
- 能说明其面积、时序或功耗影响；
- 设计决定和已知限制记录在中文文档中。

## 2. 总体设计原则

- 正确性和可验证性优先于功能数量和频率指标。
- 采用自上而下设计：总体边界、微架构、模块接口、RTL、验证逐层推进。
- 核心数据通路保持工艺无关，SMIC 28nm 工艺库只用于综合和时序分析。
- 指令、数据、控制和副作用必须具有明确的流水级归属。
- 每一级流水状态都使用 `valid` 表示是否包含真实指令。
- 错误路径或无效指令不得产生寄存器写、存储器写和协处理器请求。
- 外部总线协议不直接侵入五级流水，后续通过适配器连接 SRAM、Cache 或 AXI。
- 为未来协处理器扩展保留架构位置，但当前不实现 NPU 数据通路。

## 3. 首个可运行版本 v0.1

### 3.1 微架构

- 32 位数据宽度和地址宽度；
- 单发射、顺序执行；
- `IF / ID / EX / MEM / WB` 五级流水；
- 指令和数据通路分离，避免单端口存储器造成结构冒险；
- 分支在 EX 阶段判断，采用静态不跳转策略；
- 支持 EX/MEM 和 MEM/WB 到 EX 的数据前递；
- load-use hazard 暂停一个周期；
- 单时钟域，核心内部采用同步复位；
- 复位入口地址使用参数 `RESET_VECTOR` 配置。

### 3.2 指令范围

v0.1 首先实现 RV32I 中构成基本整数程序所需的指令：

- 整数寄存器运算：`ADD、SUB、SLL、SLT、SLTU、XOR、SRL、SRA、OR、AND`；
- 整数立即数运算：`ADDI、SLTI、SLTIU、XORI、ORI、ANDI、SLLI、SRLI、SRAI`；
- 高位立即数：`LUI、AUIPC`；
- 条件分支：`BEQ、BNE、BLT、BGE、BLTU、BGEU`；
- 无条件跳转：`JAL、JALR`；
- 访存：`LB、LH、LW、LBU、LHU、SB、SH、SW`。

v0.1 的测试程序只使用自然对齐的取指和数据访问。非对齐访问、非法指令、`ECALL` 和 `EBREAK` 的精确异常行为在异常单元阶段补充。在此之前，不把 v0.1 宣称为完整的特权架构兼容实现。

### 3.3 首版运行环境

- 使用独立的指令存储器模型和数据存储器模型；
- 首次流水集成采用无等待或固定延迟的存储器响应；
- 通过手写汇编和小型裸机程序进行验证；
- 使用退休跟踪信息观察每条有效指令的 PC、指令字和写回结果。

## 4. 后续能力分层

| 阶段 | 新增能力 |
|---|---|
| v0.2 | 可变存储器延迟、非法指令和非对齐访问处理、Machine Mode 基本 CSR、精确异常 |
| v0.3 | 中断、迭代式 RV32M 乘除法单元、多周期流水暂停 |
| v0.4 | SRAM/AXI 适配器、ISA 回归和更完整的差分验证 |
| v1.0 | SpyGlass、VCS、DC、Formality、PT 和门级仿真的可重复闭环 |
| 后续扩展 | 阻塞式协处理器命令接口、NPU、Cache 和性能优化 |

版本号表示研发里程碑，不表示对外发布的软件版本。

## 5. 当前明确不做的内容

- 乱序执行、超标量和多发射；
- 多核和缓存一致性；
- MMU、虚拟内存和 Linux 启动；
- 动态分支预测；
- 一级 Cache；
- 非阻塞访存和多个在途访存请求；
- 浮点和向量扩展；
- 当前阶段的 NPU RTL；
- 在核心内部直接实现 AXI 协议；
- 在功能稳定前追求极限频率或过早进行局部 PPA 优化。

上述内容是当前项目的范围控制，不代表架构永久禁止扩展。

## 6. 首版完成标准

v0.1 只有同时满足以下条件才算完成：

- 规定的指令均有定向测试；
- RISC-V 官方架构测试集中与当前已实现 ISA 子集匹配的测试全部通过；
- 官方测试所使用的测试集版本、配置、工具链版本和结果必须记录，不能通过临时忽略失败用例来宣称通过；
- 数据冒险、load-use、分支冲刷和 store 副作用均有关键波形；
- 错误路径指令不会写寄存器或数据存储器；
- 退休顺序与参考执行结果一致；
- RTL 不存在锁存器、多驱动和不可综合语句；
- VCS 回归稳定通过；
- SpyGlass 不存在未解释的高严重级问题；
- DC 可以完成综合，且不存在未约束的关键时序路径；
- 关键设计决定、限制和验证结果已记录。

官方架构测试主要验证指令的架构语义，不能替代处理器内部的定向测试、流水冒险测试、随机延迟测试、断言和波形检查。
