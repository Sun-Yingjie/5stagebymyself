## 1. 冻结性质

本次冻结是五级流水 core 的**功能 RTL 基线**，用于：

- 保留进入 trap/CSR/RV32M 修改前的稳定参考点；
- 支撑后续回归和 bug bisect；
- 固定当前 37 条指令、流水控制和存储接口行为；
- 形成可复现的 core directed verification 入口。

它不表示：

- 完整 RV32I 或 RISC-V Profile 认证；
- ACT4 certification tests 已完成；
- 参考模型差分已经完成；
- VCS、DC、Formality、PrimeTime 或门级 signoff 已完成；
- trap/CSR、interrupt 或 RV32M 已实现。

因此标签使用 `v0.1-rtl-baseline`，而不是把当前结果包装成完整 ISA 或 ASIC signoff 版本。

## 2. 冻结范围

### 2.1 微架构

- `XLEN=32`；
- 单 hart、单发射、顺序执行；
- `IF / ID / EX / MEM / WB` 五级流水；
- 每级流水寄存器使用 `valid`；
- EX/MEM 和 MEM/WB 到 EX 的前递；
- WB→ID 同周期旁路；
- load-use 插入一个 bubble；
- branch/JAL/JALR 在 EX 产生 redirect；
- 指令和数据通道分离；
- 每通道最多一笔在途事务；
- 阻塞式 valid-ready 请求/响应接口；
- WB 统一退休；
- 协处理器端口在 v0.1 禁用。

### 2.2 指令

冻结 37 条 RV32I 程序子集：

```text
ADD SUB SLL SLT SLTU XOR SRL SRA OR AND
ADDI SLTI SLTIU XORI ORI ANDI SLLI SRLI SRAI
LUI AUIPC
BEQ BNE BLT BGE BLTU BGEU
JAL JALR
LB LH LW LBU LHU
SB SH SW
```

## 5. 已知限制与未完成门禁

以下项目明确不属于本次功能基线通过项：

- `FENCE、ECALL、EBREAK`；
- 精确 illegal/misaligned/access-fault trap；
- Zicsr 和 Machine Mode；
- Machine interrupt 和 `MRET/WFI`；
- RV32M；
- ACT4；
- Spike/Sail 等参考模型差分；
- 任意长等待和广泛随机 backpressure；
- VCS 正式回归；
- DC、Formality、PrimeTime 和门级仿真。

SpyGlass W528 对应的 `mem_exception` 未消费问题将在 v0.2 trap commit 路径中关闭；不能在后续版本中永久豁免。

## 6. 进入下一版本的规则

从本标签之后开始修改 trap、CSR、MDU 或 interrupt RTL。每个增量必须：

1. 保持本版本 11/11 unit 和 7/7 core 回归通过；
2. 为新增架构行为增加 directed test；
3. 不改变已冻结的 valid-ready 和一次提交语义，除非先更新上位契约；
4. 发现回退时可使用 `v0.1-rtl-baseline` 进行 bisect 和波形对照。

下一版本首先冻结统一 trap/commit、architectural next PC、长延迟操作和副作用门控合同，然后增量实现完整 RV32I、Zicsr/M-mode、RV32M 和 interrupt。
